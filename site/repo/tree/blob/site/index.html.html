108 <div>
109 <p class="eyebrow">why gossip</p>
110 <h2>One witness layer for many kinds of infrastructure.</h2>
111 </div>
112 <div class="copy-block">
113 <p>
114 Nodes relay transactions, health receipts, browser receipts, compute receipts, heartbeat
115 observations, storage proofs, domain leases, swap records, and block approvals. State
116 changes finalize only after enough independent witnesses agree.
117 </p>
118 <p>
119 The live launch is federated: the Gossip Team operates the first machines, admits new
120 validators, and expands through code contributions, infrastructure commitments, and
121 direct operator coordination.
122 </p>

191 <section id="economics" class="section split economics">
192 <div>
193 <p class="eyebrow">implemented economics</p>
194 <h2>Pegged accounting that already lives in the ledger.</h2>
195 </div>
196 <div class="copy-block">
197 <p>
198 All four resource tokens use six decimal places in the Rust implementation. Genesis can
199 assign starting balances for HT, CT, ST, and DNS independently, and transfers keep each
200 asset in its own balance.
201 </p>
202 <ul class="fact-list">
203 <li>Health checks start at 1 HT; browser checks start at 2 HT and scale with steps, runtime, video, and trace capture.</li>
204 <li>Compute jobs spend CT based on job cost and effective replication across validators.</li>
205 <li>Storage contracts prepay ST and reward hosts per 64 MiB-second after validator proof consensus.</li>
206 <li>DNS leases cost 1 DNS per subdomain-second and resolve to public raw storage contracts.</li>
207 <li>Swap adapters support signed HT quotes against USDC and USDT settlement rails.</li>
208 </ul>
209 </div>

212 <section id="compute" class="section compute-band">
213 <div class="section-heading">
214 <p class="eyebrow">resource lifecycle</p>
215 <h2>Submit. Witness. Settle. Publish.</h2>
216 </div>
217 <div class="flow">
218 <div>
219 <span>1</span>
220 <h3>Submit</h3>
221 <p>Users sign checks, browser journeys, compute jobs, storage contracts, swaps, or DNS leases.</p>
222 </div>
223 <div>
224 <span>2</span>
225 <h3>Witness</h3>
226 <p>Validators and delegated agents execute work, observe heartbeats, prove storage, and sign receipts.</p>
227 </div>
228 <div>
229 <span>3</span>
230 <h3>Settle</h3>
231 <p>Blocks apply balances across HT, CT, ST, and DNS while rejecting conflicting evidence.</p>
232 </div>
233 <div>
234 <span>4</span>
235 <h3>Publish</h3>
236 <p>Artifacts, monitor state, storage routes, and lease bindings remain as verifiable public records.</p>
237 </div>
238 </div>
