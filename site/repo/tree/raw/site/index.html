why gossip

One witness layer for many kinds of infrastructure.

Nodes relay transactions, health receipts, browser receipts, compute receipts, heartbeat observations, storage proofs, domain leases, swap records, and block approvals. State changes finalize only after enough independent witnesses agree.

The live launch is federated: the Gossip Team operates the first machines, admits new validators, and expands through code contributions, infrastructure commitments, and direct operator coordination.

resource markets

Four pegged resource markets, one receipt layer.

HT

HT Credits

Base credit for health checks, browser journeys, monitor budgets, validator rewards, and swap workflows.

CT

Compute Token

Pays for sharded compute, WASI execution, replication, consensus receipts, and returned artifacts.

ST

Storage Token

Prepays storage contracts and rewards hosts that keep encrypted or public raw content available.

DNS

DNS Token

Leases names, renews routes, and binds public hostnames to active storage-backed content.

implemented economics

Pegged accounting that already lives in the ledger.

All four resource tokens use six decimal places in the Rust implementation. Genesis can assign starting balances for HT, CT, ST, and DNS independently, and transfers keep each asset in its own balance.

  • Health checks start at 1 HT; browser checks start at 2 HT and scale with steps, runtime, video, and trace capture.
  • Compute jobs spend CT based on job cost and effective replication across validators.
  • Storage contracts prepay ST and reward hosts per 64 MiB-second after validator proof consensus.
  • DNS leases cost 1 DNS per subdomain-second and resolve to public raw storage contracts.
  • Swap adapters support signed HT quotes against USDC and USDT settlement rails.

resource lifecycle

Submit. Witness. Settle. Publish.

1

Submit

Users sign checks, browser journeys, compute jobs, storage contracts, swaps, or DNS leases.

2

Witness

Validators and delegated agents execute work, observe heartbeats, prove storage, and sign receipts.

3

Settle

Blocks apply balances across HT, CT, ST, and DNS while rejecting conflicting evidence.

4

Publish

Artifacts, monitor state, storage routes, and lease bindings remain as verifiable public records.
