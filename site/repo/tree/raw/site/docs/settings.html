settings

Commands, fields, defaults, and limits

This page documents the current Rust implementation settings. Amounts use six decimal places internally, where 1.0 token equals 1_000_000 micro-units.

Command groups

Group Subcommands Purpose
wallet create, address, show Create and inspect signing identities.
genesis create Create the chain ID, validator set, treasury, block time, receipt threshold, and starting balances.
node run Run a validator node with HTTP control and gossip endpoints.
tx Transfers, health checks, browser checks, compute jobs, monitors, storage, and DNS operations. Build and submit signed ledger transactions.
swap quote, lock, cancel, settle Quote and settle HT against configured settlement adapters.
monitor hash-token Hash heartbeat URL secrets before placing them in monitor specs.
browser validate-package, hash-package Validate and hash browser journey packages.
storage bundle, module-ref, host, prove Build bundles, serve chunks, create WASI module refs, and submit storage proofs.
agent run Run a delegated health, browser, and compute worker.

Heartbeat auth supports secret_url, delegated_key, or dual. Signal policies support success only, start plus success, or start plus success plus fail. Miss policies support record-only, validator confirmation, or delegated agent confirmation with optional region diversity.
